# HLaunch
开发一款安卓应用，定位为HTML 微应用启动器 & 管理器，核心目标是让用户可在手机端轻量化运行、管理自定义 HTML 文件，并支持 Git 同步能力，具体功能拆解如下：
一、核心运行能力
作为 HTML 文件的专属启动器，支持同时管理 / 运行多个 HTML 文件，后台多任务逻辑参考微信小程序（如保留运行状态、切换不中断、支持多窗口 / 多标签切换）；
内置轻量浏览器内核（如 WebView），确保 HTML 文件能完整运行（支持 JS/CSS/ 本地资源加载）。
二、HTML 文件管理能力
创建功能：提供文本输入框，支持用户粘贴 / 输入 HTML 代码，保存为独立的 HTML 文件（支持自定义文件名）；
文件来源 & 标识：
支持添加本地 HTML 文件（从手机存储选择）；
支持添加 Git 仓库链接，自动拉取 / 同步仓库内的 HTML 文件；
所有文件需清晰标识来源（如 “本地文件”“Git 同步文件” 标签区分）；
文件关联操作：支持将本地创建 / 导入的 HTML 文件手动添加到指定 Git 仓库（提交 / 推送）。
三、Git 同步能力
针对 Git 来源的文件：支持手动 / 自动检测仓库更新，一键拉取最新版本；
针对本地转 Git 的文件：支持编辑后提交、推送至对应仓库；
基础 Git 配置：支持输入仓库地址、账号令牌（如 GitHub PAT）、分支选择等。
四、应用自身更新机制
应用不依赖应用商店更新，通过 Git 仓库检测更新：
预设存放安装包（APK）的 Git 仓库地址；
应用启动 / 手动触发时，检测仓库内最新版本号与本地版本对比；
检测到更新后，提示用户下载并安装最新 APK。
五、核心使用场景（补充，强化需求逻辑）
用户通过 AI 生成 HTML 代码 → 粘贴到应用内创建 HTML 文件 → 直接在应用内运行预览 → 如需保存 / 同步，可关联 Git 仓库推送 → 后续在其他设备打开应用，拉取 Git 仓库即可同步使用该 HTML 文件。
总结
核心定位是移动端 HTML 微应用管理器，核心能力为 “运行 + 创建 + Git 同步”；
关键细节：文件来源标识、微信小程序式多任务、应用自身通过 Git 更新；
核心场景：适配 AI 生成 HTML 代码的快速落地、轻量化运行需求。
